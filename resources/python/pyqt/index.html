    PyQt

    Introduction text here ...

    PyQt is originally build in C then ported to python. Lots of documentations and problems solving are in C. Hence PyQt has becoming more popular in Python a lot of resources in problems handling are also available.

    Import Dependencies

    $ pip install PyQt6

    Qt Designer

    Pre-requisition: have PyQt installed in the virtual environment or in the project

    • PyQt5: use $ pip install pyqt5-tools
    • PyQt6: use $ pip install pyqt6-tools

    Open Qt Designer by using the command qt6-tools designer (or qt5-tools designer) in terminal. Noted that you can only open Qt Designer from terminal when working in a virtual environment.

    Another way to open Qt Designer, in your IDE, go to venv/lib/python3.8/site-packages/qt6_applications/Qt/bin. Open the folder and double-click the Designer.exe (on Windows) or Designer.app (on macOS)

    Note

    I find that QT Designer in this method not nice to working with. The GUI and toolbars aren't combined in a single window.

    (Optional) Qt Designer for Mac (Qt Creator)

    Download and Install

    • Go to download page.
    • Download open source
    • Create an account and verify (or later)
    • If you haven't Xcode, download it
    • Open the .dmg file
    • Install

    The installment could take an hour depending on your internet speed and the server download speed. On my side, it took almost 2 hours to download 5.32 Gb.

    Open and Create a new Qt project

    • Open Qt Designer
    • Go to File/New File.... This method will allow to create a .ui file for the project and save it to our project folder
    • In Files and CLasses select Qt then Qt Designer Form and click the Choose... button
    • In Choose a Form Template, select the template you want such as Main Window and click the Continue button
    • Enter a File name and the Path to save, and click the Continue button
    • Next, you can add version control (if any) and then click the Done button

    In my opinion, Qt Creator looks better than Qt Designer and run in parallel to the IDE, unlike the Qt Designer.

    QToolBars

    import sys
from PyQt6.QtCore import *
from PyQt6.QtGui import *
from PyQt6.QtWidgets import *


class Demo(QMainWindow):
    def __init__(self, parent=None):
        super(Demo, self).__init__(parent)
        layout = QVBoxLayout()
        tb = self.addToolBar("File")

        new = QAction(QIcon("images/new.png"), "new", self)
        tb.addAction(new)

        o = QAction(QIcon("images/open.png"), "open", self)
        tb.addAction(o)
        save = QAction(QIcon("images/save.png"), "save", self)
        tb.addAction(save)
        tb.actionTriggered[QAction].connect(self.on_click_button)
        self.setLayout(layout)
        self.setWindowTitle("toolbar demo")

    def on_click_button(self, a):
        print("pressed tool button is", a.text())


def main():
    app = QApplication(sys.argv)
    window = Demo()
    window.show()
    sys.exit(app.exec())


if __name__ == '__main__':
    main()

    QDockWidget

    import sys

from PyQt6.QtCore import *
from PyQt6.QtWidgets import *


class Demo(QMainWindow):
    def __init__(self, parent=None):
        super(Demo, self).__init__(parent)
        layout = QHBoxLayout()
        bar = self.menuBar()
        file = bar.addMenu('File')
        file.addAction('New')
        file.addAction('Save')
        file.addAction('quit')

        self.items = QDockWidget('Dockable', self)

        self.listWidget = QListWidget()
        self.listWidget.addItem('Item1')
        self.listWidget.addItem('Item2')
        self.listWidget.addItem('Item3')
        self.listWidget.addItem('Item4')

        self.items.setWidget(self.listWidget)
        self.items.setFloating(False)
        self.setCentralWidget(QTextEdit())
        self.addDockWidget(Qt.DockWidgetArea.BottomDockWidgetArea, self.items)

        self.setLayout(layout)
        self.setWindowTitle('Dock')


if __name__ == '__main__':
    app = QApplication(sys.argv)
    demo = Demo()
    demo.show()
    sys.exit(app.exec())
